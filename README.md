# Vote Ray Backend

A Rust-based voting system backend with layered architecture using SeaORM.

## Architecture

The project follows a clean architecture pattern with the following layers:

- **Business Layer** (`src/business/`): Entity classes and business logic
- **Database Layer** (`src/database/`): SeaORM entities and repositories
- **REST Layer** (`src/rest/`): API endpoints and DTOs

## Technology Stack

- **Web Framework**: Axum
- **ORM**: SeaORM
- **Database**: PostgreSQL
- **Documentation**: Swagger/OpenAPI with utoipa
- **Serialization**: Serde

## Entities

### Candidate
- `id`: UUID (primary key, generated by frontend)
- `name`: String

### Pledge
- `id`: UUID (primary key, generated by frontend)
- `name`: String
- `description`: String (longer text)

## API Endpoints

### Candidates (api/v1/candidates)
- `POST /api/v1/candidates` - Create a new candidate
- `GET /api/v1/candidates` - Get all candidates
- `GET /api/v1/candidates/{id}` - Get a specific candidate
- `PUT /api/v1/candidates/{id}` - Update a candidate
- `DELETE /api/v1/candidates/{id}` - Delete a candidate

### Pledges (api/v1/pledges)
- `POST /api/v1/pledges` - Create a new pledge
- `GET /api/v1/pledges` - Get all pledges
- `GET /api/v1/pledges/{id}` - Get a specific pledge
- `PUT /api/v1/pledges/{id}` - Update a pledge
- `DELETE /api/v1/pledges/{id}` - Delete a pledge

## Setup

### Option 1: Local PostgreSQL
1. Set up PostgreSQL database
2. Create a `.env` file with:
   ```
   DATABASE_URL=postgresql://vote_ray:your_password@localhost:5432/vote_ray
   ```

### Option 2: Podman PostgreSQL (Recommended)
1. Start PostgreSQL with Podman:
   ```bash
   podman run --name postgres-vote \
     -e POSTGRES_DB=vote_ray \
     -e POSTGRES_USER=vote_ray \
     -e POSTGRES_PASSWORD=secure_password \
     -p 5432:5432 \
     -d postgres:15
   ```
2. Create a `.env` file:
   ```
   DATABASE_URL=postgresql://vote_ray:secure_password@localhost:5432/vote_ray
   ```

### Option 3: Nix Development Environment
1. Start PostgreSQL with Nix:
   ```bash
   nix shell nixpkgs#postgresql --command "initdb -D /tmp/pgdata && pg_ctl -D /tmp/pgdata -l logfile start"
   ```
2. Create database:
   ```bash
   createdb vote_ray
   ```

3. Run SeaORM migrations:
   ```bash
   cd migration && cargo run -- up
   ```

4. Run the application:
   ```bash
   cargo run
   ```

## SeaORM Features Used

- **Entity Definitions**: Auto-generated from struct definitions
- **Repository Pattern**: Clean abstraction over database operations
- **Active Record Pattern**: Easy entity manipulation
- **Migrations**: Database schema management
- **Type Safety**: Compile-time query validation
